# Decision: Local-First Architecture
## ADR-005

We decided that ARES must operate entirely locally without requiring network access, cloud services, or external databases.

Context:
Developer tools that require cloud connectivity face adoption friction, data sovereignty concerns, and availability risks. Repository memory is inherently local — it describes code that lives on disk.

Decision:
All ARES functionality runs on the developer's machine. The knowledge graph is stored in .ares/ares.db (SQLite). No network calls during ingestion, querying, or validation. The MCP server runs on stdio, not HTTP.

Consequences:
- Zero cloud infrastructure cost.
- Works offline, in air-gapped environments, and behind corporate firewalls.
- Data never leaves the developer's machine.
- Future cloud sync can be layered on top without changing the core.

This decision satisfies REQ-MEMORY-001.
This decision drives crates/ares-knowledge-graph/src/store.rs and crates/ares-mcp/src/main.rs.

Status: Accepted
