# Decision: Incremental Capture via File Watcher
## ADR-006

We decided to support incremental ingestion triggered by a file watcher, rather than requiring full rescans.

Context:
Re-ingesting a repository with 10k+ files takes several seconds. For an IDE extension to provide real-time intelligence, the knowledge graph must update near-instantly when a developer saves a file.

Decision:
The ares-mcp server and CLI will support passing a specific list of changed files. The ingestion graph builder will filter its walkdir results against this list, and only upsert nodes and edges for the modified files.

Consequences:
- Real-time updates in VS Code.
- No need to hold file watcher state inside the core engine; the external client (VS Code, CLI watcher) triggers the update.
- Edge removal is not currently handled by simple upserts, necessitating a future "simulate" or "prune" mechanism.

This decision satisfies REQ-INGEST-004.
This decision drives crates/ares-ingestion/src/graph.rs and crates/ares-mcp/src/main.rs.

Status: Accepted
