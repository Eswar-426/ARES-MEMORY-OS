# Decision: Foreign Key Enforcement
## ADR-015

We decided to enforce PRAGMA foreign_keys = ON at the SQLite database layer.

Context:
In early prototype stages, ARES suffered from dangling edge references because an extractor would create an edge to an entity that had not yet been ingested, or a node was deleted without cascading edge deletions.

Decision:
Foreign keys are strictly enabled. The `graph_relationships` table has `source_entity` and `target_entity` fields that enforce `REFERENCES graph_entities(id) ON DELETE CASCADE`. If an extractor attempts to create an edge to an entity that doesn't exist, it must synthesize the missing node before creating the edge, or the SQLite transaction will fail.

Consequences:
- Graph integrity is mathematically guaranteed.
- Zero dangling edges.
- Extractors (like TestResolution and Ownership) must gracefully handle "forward declarations" of nodes.

This decision satisfies REQ-MEMORY-001.
This decision drives crates/ares-knowledge-graph/src/store.rs.

Status: Accepted
