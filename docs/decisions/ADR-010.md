# Decision: Markdown Intelligence Extraction
## ADR-010

We decided to extract knowledge edges by parsing markdown documents for explicit semantic tokens rather than using natural language processing.

Context:
To link requirements and decisions to code, we need a reliable way to map text to graph entities. LLM-based extraction is slow, non-deterministic, and breaks the local-first requirement.

Decision:
The `MarkdownIntelligenceExtractor` scans `.md` files for explicit tokens (e.g., `REQ-xxx`, `ADR-xxx`) and canonical code paths. If a decision document contains the string `crates/ares-mcp/src/main.rs`, a `Drives` edge is deterministically created.

Consequences:
- Extraction is instantaneous and fully deterministic.
- Forces developers to use explicit, searchable identifiers in their documentation, improving overall hygiene.

This decision satisfies REQ-INGEST-003 and REQ-MEMORY-010.
This decision drives crates/ares-ingestion/src/extractors/markdown.rs.

Status: Accepted
