# Decision: Deterministic Gap Engine
## ADR-004

We decided that knowledge gap detection must be fully deterministic, computed during graph construction rather than via probabilistic or LLM-based analysis.

Context:
Non-deterministic gap detection would produce inconsistent results across runs, undermining memory quality certification. Gaps are part of memory integrity and must be reproducible.

Decision:
Compute gaps during every full ingest and every incremental ingest. Gap types are derived from graph structure: missing edges between expected node type pairs. No heuristics, no ML, no LLM involvement.

Consequences:
- Gap detection is 100% reproducible across identical repository states.
- No external service dependencies for gap analysis.
- Gap results can be cached and diffed between ingestion runs.
- False positive rate is 0% for structural gaps.

This decision satisfies REQ-MEMORY-003.
This decision drives crates/ares-gap-engine/src/lib.rs and crates/ares-ingestion/src/gap_generator.rs.

Status: Accepted
