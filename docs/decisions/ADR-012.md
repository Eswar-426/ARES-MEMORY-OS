# Decision: Bounded Memory via Batch Commits
## ADR-012

We decided to enforce a strict upper bound on memory consumption during ingestion by using batch commits.

Context:
Loading a 100k-node graph into RAM causes ARES to exceed its 100MB memory budget. To be enterprise-ready, memory scaling must be O(1) relative to repository size.

Decision:
The `upsert_batch` method in `KnowledgeGraphStore` buffers exactly 2500 events (nodes/edges) at a time. Once the threshold is hit, the transaction is committed, dropping the Rust structures from memory and continuing the stream.

Consequences:
- Memory footprint remains flat regardless of repository size.
- A failure during batch 45 rolls back only batch 45, while batches 1-44 remain safe.

This decision satisfies REQ-INGEST-005.
This decision drives crates/ares-knowledge-graph/src/store.rs.

Status: Accepted
