# Decision: Deterministic Test Resolution
## ADR-009

We decided to resolve tests deterministically using file path and naming heuristics rather than dynamic code execution or coverage reports.

Context:
To link code to tests, ARES needs to establish `ValidatedBy` edges. Running test suites to extract coverage data is extremely slow and requires executing untrusted code, which violates ARES safety and performance constraints.

Decision:
The `TestResolutionEngine` uses language-specific conventions to match source files to test files (e.g., `foo.rs` -> `foo_test.rs` or `tests/foo.rs`). If a test file matching the pattern exists in the graph, the edge is created.

Consequences:
- Instant test resolution without executing code.
- 100% deterministic results.
- Misses some integration tests that do not follow naming conventions, but ensures high precision for unit tests.

This decision satisfies REQ-INTEL-003.
This decision drives crates/ares-ingestion/src/extractors/tests.rs.

Status: Accepted
