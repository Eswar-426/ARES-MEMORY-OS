# Decision: MCP over Stdio Transport
## ADR-008

We decided to expose ARES intelligence using the Model Context Protocol (MCP) over standard input/output (stdio), rather than HTTP or gRPC.

Context:
IDE extensions and AI agents need a standardized way to request context. HTTP servers require port management, firewall configuration, and background daemon lifecycles.

Decision:
Implement the MCP specification using stdio transport. The VS Code extension spawns the `ares-mcp` binary as a child process and communicates via JSON-RPC over stdin/stdout.

Consequences:
- Zero configuration required for the end user.
- Lifecycle is intrinsically tied to the client process.
- Stdio transport guarantees security since only the parent process can communicate with it.

This decision satisfies REQ-MEMORY-006.
This decision drives crates/ares-mcp/src/main.rs.

Status: Accepted
