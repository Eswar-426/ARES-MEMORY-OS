# Decision: Repository Event Compression
## ADR-014

We decided to compress repository timeline events to minimize the growth of the SQLite database.

Context:
If a user saves a file 50 times in one hour, generating 50 discrete `RepositoryEvent` records for identical `Modify` operations wastes storage and query bandwidth during evolution analysis.

Decision:
The Evolution Engine performs run-length encoding style compression. Consecutive events of the same type on the same entity within a specific time window are merged into a single event with an incremented operation count.

Consequences:
- Drastically reduced timeline storage overhead.
- "Evolution" MCP queries return cleaner, more meaningful histories instead of spam.

This decision satisfies REQ-MEMORY-005.
This decision drives crates/ares-memory-evolution/src/lib.rs.

Status: Accepted
