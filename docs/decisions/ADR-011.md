# Decision: CODEOWNERS-based Ownership
## ADR-011

We decided to use GitHub's CODEOWNERS standard to establish `OwnedBy` edges in the graph.

Context:
Governance policies and gap analysis require identifying unowned code. To avoid creating a proprietary configuration format for ARES, we must leverage an existing standard.

Decision:
The `OwnershipExtractor` parses `CODEOWNERS` and `.github/CODEOWNERS` files. It translates the glob patterns and owner identities into `Owner` nodes and `OwnedBy` edges connecting the code artifacts to their respective owners.

Consequences:
- Zero adoption friction for repositories already using GitHub.
- Pattern matching natively supports wildcard fallbacks (e.g., `* @team`).

This decision satisfies REQ-MEMORY-015.
This decision drives crates/ares-ingestion/src/extractors/ownership.rs.

Status: Accepted
