# Decision: WAL Journal Mode for Concurrency
## ADR-013

We decided to configure SQLite with Write-Ahead Logging (WAL) journal mode.

Context:
ARES requires concurrent operations. For example, while `ares ingest .` is running and performing a large batch transaction, the VS Code extension (`ares-mcp`) must still be able to answer "Why Exists" and "Impact" queries without blocking. Standard SQLite locking blocks readers during a write transaction.

Decision:
Execute `PRAGMA journal_mode = WAL;` and `PRAGMA synchronous = NORMAL;` upon connection establishment. 

Consequences:
- Readers do not block writers, and writers do not block readers.
- Greatly improved read throughput during heavy ingestion workloads.

This decision satisfies REQ-MEMORY-001.
This decision drives crates/ares-knowledge-graph/src/store.rs.

Status: Accepted
