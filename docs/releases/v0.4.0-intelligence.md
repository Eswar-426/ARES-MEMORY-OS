# Release v0.4.0-intelligence

**Date:** July 2, 2026
**Commit:** `0b4c97f`

This release represents a massive milestone for ARES Memory OS. It marks the transition from a raw data pipeline into a cohesive, senior-engineer-level **Repository Intelligence System**. The raw VS Code output channel has been entirely replaced by a stable, dynamic webview UI that flawlessly renders deep codebase insights.

## Highlights

* **Unified Narrative Engine:** A shared `narrative.rs` compositor that eliminates duplicate logic, standardizing all intelligence outputs into professional Purpose, History, Architecture, and Ownership sections.
* **Semantic Evidence Model:** The UI intelligently flattens and deduplicates evidence (folders, dependencies, dependents, contributors), ensuring no redundancy between the raw evidence list and the written narrative.
* **Intent Extraction:** Automatically parses Git commit bodies for keywords (e.g., `Decision:`, `Tradeoff:`) and weaves architectural intent directly into the knowledge graph.
* **Multi-Hop Git Traversal:** Deep traversal of file history mapping structural changes to individual commits and authors.
* **Drift Intelligence:** Advanced calculations to measure Knowledge Debt and Structural Stability over time based on modification frequency and contributor churn.
* **Traceability:** Bidirectional tracing of dependencies to architectural decisions.
* **Test-Aware Impact Analysis:** Excludes test dependencies (`is_test = true`) from blast radius and drift calculations to drastically reduce noise and focus strictly on core architecture.
* **VS Code Query Panel:** A native, deeply integrated UI replacing raw JSON outputs.
* **Confidence Engine:** Bulletproof semantic confidence scoring (0-100) rendered with a dedicated checklist UI.

## Technical Fixes
* Eradicated the Webview `NaN%` Confidence bug via strict JSON parsing and DOM updates.
* Implemented aggressive `setTimeout` cache-busting to prevent VS Code from loading stale webview HTML across sessions.
* Decommissioned legacy widgets (e.g., the old discrete Drift panel) in favor of the new, unified markdown stream.
* Repaired broken Git edge persistence (`AuthoredBy`, `Owner`) in the `ares-cli` ingest process.
