## 3. Structural Bug Fixes

The certification process exposed several critical issues inside the SQLite schema constraints which were previously suppressing extraction silently:

1. **V38 Migration (Edge Expansion):** The original SQLite `CHECK` constraint on `edge_type` only allowed governance and architectural edges (`depends_on`, `owns`, etc.). We expanded this to include factual edges (`touches`, `maintains`, `authored_by`, `released_in`, `contributed_to`).
2. **V39 Migration (Node Expansion):** The original SQLite `CHECK` constraint on `node_type` was strictly limited to code and documentation elements. We expanded this to include factual event nodes (`commit`, `branch`, `release`, `person`).

Both constraints were successfully deployed, clearing the bottlenecks and allowing the ingestion of massive memory graphs without database panics or deadlocks.
